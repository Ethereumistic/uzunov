
## Routing (TanStack Start)
- Routes are files in `app/routes/`. Filename = URL path.
- Every route exports `const Route = createFileRoute('/path')(...)`.
- Nested layouts: `_layout.tsx` wraps child routes.
- Root route `__root.tsx` must expose `QueryClient` in context.
- Lazy data: use `loader` in route options + `Route.useLoaderData()`.
- Navigate: `useNavigate()` hook or `<Link to="/path">`.

## Convex Integration
- **Never import from `convex/_generated/` directly in components** — always use `api` object: `import { api } from '../convex/_generated/api'`.
- Query (reactive, real-time): `useQuery(api.file.fnName)`
- Mutation: `useMutation(api.file.fnName)`
- Convex + React Query: `convexQuery(api.file.fnName, args)` as queryOptions.
- Backend functions live in `convex/*.ts`. After editing, `npx convex dev` must be running to sync.
- Schema changes require updating `convex/schema.ts` + rerunning `npx convex dev`.
- Validators: use `v` from `"convex/values"` — `v.string()`, `v.number()`, `v.id("tableName")`.

```ts
// convex/items.ts
import { query, mutation } from "./_generated/server";
import { v } from "convex/values";

export const list = query({ handler: async (ctx) => ctx.db.query("items").collect() });
export const create = mutation({
  args: { title: v.string() },
  handler: async (ctx, args) => ctx.db.insert("items", { title: args.title }),
});
```

## Key Patterns

### Fetch data in a route loader (SSR-safe)
```ts
export const Route = createFileRoute('/projects')({
  loader: ({ context: { queryClient } }) =>
    queryClient.ensureQueryData(convexQuery(api.projects.list, {})),
  component: () => {
    const projects = useQuery(api.projects.list);
    return <div>{projects?.map(p => <div key={p._id}>{p.title}</div>)}</div>;
  },
});
```

### Mutation in component
```ts
const create = useMutation(api.projects.create);
await create({ title: "New Project" });
```

## Pitfalls
- `routeTree.gen.ts` is auto-generated by TanStack Router — never edit it.
- TanStack Start is RC — avoid experimental APIs; check console for deprecation warnings.
- Convex queries are **live/reactive** by default. Don't poll manually.
- `_generated/` rebuilds on `npx convex dev` — if types are stale, restart convex dev.
- Do not use `<form>` HTML elements — use onClick/onChange handlers with `useMutation`.
- Environment variable for Convex URL: `VITE_CONVEX_URL` in `.env.local`.

## Code Style
- TypeScript strict mode. No `any` unless unavoidable.
- Prefer named exports for components.
- Co-locate route-specific components inside the route file unless reused.
- Tailwind for styling (utility-first, no inline styles).
